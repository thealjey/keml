Just like [on:\*](../event/on-colon.md) is the output channel for event actions,
so too are the `result` and `error` attributes the output channels for result
actions.

Result actions can be defined on any element that can receive server results.
This means that the element is either sending requests to the server, or is
directly pushed results into by the server (e.g. [SSE](../sse.md)).

They are defined like so:

- `result="all good"` → actions generated by successful status codes
- `error="something amiss"` → actions generated by unsuccessful status codes

They are identical in every way and only filter by the response status code.

---

The elements that receive server results, regardless of whether or not they have
`result` and/or `error` attributes, will emit events on themselves upon receipt
of those results:

- `result` → successful status code
- `failure` → unsuccessful status code (if you're wondering why not `error`, the
  reason is simply because `error` is already an existing reserved browser event
  type)
