The point is not the grid, but the fact that it is a completely normal div element, with no grid libraries or any kind of special rendering needed.

This approach does come with certain limitations, however, since browsers do have a hard limit on the total scrollable layout height beyond which layout calculations and scroll positioning start to break down (e.g. 17895697px in Firefox). That means this example supports only around 350,000 rows as a practical maximum at typical row heights.

Which just happens to be exactly the number of rows we are displaying here 😜
